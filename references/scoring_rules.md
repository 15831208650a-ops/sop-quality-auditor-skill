# SOP质量评分细则

## 总分

总分100分：

| 板块 | 满分 |
|------|------|
| 基础操作指引 | 40 |
| 进阶实操提示 | 20 |
| 易错点 | 20 |
| GAAP Difference | 5 |
| Review Checklist | 12 |
| 常用网站 | 3 |

结论档位：

| 得分 | 结论 |
|------|------|
| ＜30 | 立即重写 |
| 30-50 | 大段修改 |
| 50-60 | 重点修改 |
| 60-80 | 小幅修改 |
| 80-100 | 审核通过 |

若触发0容忍问题，结论后加“必须修改”。

## PSP识别

视为PSP或审计程序章节：

- `## 汇总`
- `## 一、[工作表名称]`
- `## 二、[工作表名称]`
- 其他以中文序号开头且属于审计程序执行内容的二级标题。

不视为PSP：

- `## 使用说明`
- `## 工作流程图`
- `## PBC示例`
- `## GAAP Difference`
- `## Review Checklist`
- `## 常用网站`
- 附录、参考资料、版本记录等非程序执行章节。

## 必备章节

SOP必须包含：

- `使用说明`
- `工作流程图`
- `PBC示例`
- `基础操作指引`
- `进阶实操提示`
- `易错点`
- `GAAP Difference`
- `Review Checklist`
- `常用网站`

缺少任一必备章节：审核不通过，结论标注“必须修改”。

评分处理：

- 缺少六大评分板块之一（基础操作指引、进阶实操提示、易错点、GAAP Difference、Review Checklist、常用网站）：对应板块得分为0。
- 缺少 `使用说明`、`工作流程图` 或 `PBC示例`：列为结构完整性强制修改项；不新增单独分值，不改变六大评分板块满分结构。

## 全文通用扣分

SOP应为最终版，不得有注释或草稿标记。

| 问题 | 扣分 |
|------|------|
| Markdown注释 `<!-- -->` | 所在板块扣3分/处 |
| 代码注释 `//`、`/* */` | 所在板块扣3分/处 |
| TODO、FIXME、待补充、审核中 | 所在板块扣3分/处 |
| 全文注释总数超过2处 | 相关板块格式规范性扣完 |

## 一、基础操作指引（40分）

### 1. 标题层级准确性（10分）

标准层级：

| 层级 | 标准格式 |
|------|----------|
| 总标题 | `# [科目名称/项目名称] SOP` |
| PSP章节 | `## 一、[工作表名称]`，`## 汇总`可接受 |
| 核心步骤 | `### 第一步：[具体动作或判断]` |
| 模块标题 | `#### 基础操作指引` |
| 具体任务 | `##### 1. [具体任务名称]` |
| 细粒度操作 | `①②③`，子项 `a. b. c.` |

扣分：

| 问题 | 扣分 |
|------|------|
| 标题层级错误，导致模块归属不清 | 4分/处 |
| `###`无法识别为核心步骤 | 3分/处 |
| `#### 基础操作指引`缺失或写成正文 | 3分/处 |
| `##### 1.`写成正文或无序列表 | 2分/处 |
| 同一层级编号规则混用 | 4分 |
| 标准PSP章节大量缺失或无法识别 | 本项0分 |

### 2. 每个PSP包含基础操作指引（10分）

| 情况 | 得分或扣分 |
|------|------------|
| 所有PSP均包含 | 10分 |
| 每个PSP缺失 | 扣5分/个 |
| 超过2个PSP缺失 | 本项0分 |
| 非PSP章节缺失 | 不扣分 |

### 3. 句子简洁性（15分）

长句需同时满足：

- 超过70个中文字符，不含空格和Markdown标记。
- 含“且/并/同时/另外/此外/同步/以及”等连接词并连接两个以上独立操作，或描述三个以上独立步骤。

不视为长句：

- 70字以内。
- “若X，则Y”表达一个判断逻辑。
- 专业术语较多但动作单一。
- 外部链接、文件名、法规条文导致超长。

按短句率给分：

| 短句率 | 得分 |
|--------|------|
| >=95% | 15 |
| 90%-95% | 12 |
| 85%-90% | 9 |
| 80%-85% | 6 |
| <80% | 3 |

额外扣分：

- 超过120字的严重长句，扣1分/处，最多扣3分。
- 存在超过200字超长句，本项得分上限10分。

### 4. 格式规范性（5分）

| 要求 | 分值 |
|------|------|
| 步骤性内容用有序列表，并列内容用无序列表或同层编号 | 2 |
| 关键结论、风险警示、阈值判断可快速识别 | 2 |
| 无多余空行、异常空格、乱码、明显残留特殊字符 | 1 |

## 二、进阶实操提示（20分）

### 1. 标题层级准确性（5分）

标准层级：

| 层级 | 标准格式 |
|------|----------|
| PSP章节 | `## 一、[工作表名称]` |
| 核心步骤 | `### 第一步：[具体动作或判断]` |
| 模块标题 | `#### 进阶实操提示` |
| 进阶主题 | `##### ●[进阶主题]` |
| 一级内容 | `1. **[加粗小标题]**：[具体内容]` |
| 二级内容 | `①②③` |
| 三级内容 | `a. b. c.` |

扣分：

| 问题 | 扣分 |
|------|------|
| `#### 进阶实操提示`缺失或写成正文 | 2分/处 |
| 进阶主题未使用 `##### ●[主题]` 但可识别 | 1分/处 |
| 主题下层级混乱 | 1分/处 |
| `1.`一级内容缺少加粗小标题 | 1分/处，本项最多扣3分 |
| 进阶内容全部堆在自然段 | 本项0分 |

### 2. 内容深度与实务价值（4分）

| 情况 | 扣分 |
|------|------|
| 仅重复基础操作，无额外实务价值 | 1分/处 |
| 遗漏SOP自身结构可判断出的关键进阶要点 | 1分/处 |

不得凭外部经验主观臆测扣分。

### 3. 句子结构规范性（8分）

进阶提示中，单句超过90字且未按情景、条件或步骤拆分，视为结构不规范。

按规范句率给分：

| 规范句率 | 得分 |
|----------|------|
| >=92% | 8 |
| 88%-92% | 6 |
| 84%-88% | 4 |
| 80%-84% | 2 |
| <80% | 1 |

逻辑关系不清，扣1分/处，最多扣2分。

### 4. 格式规范性（3分）

| 要求 | 分值 |
|------|------|
| 以主题、列表或表格呈现，无大段连续段落 | 1.5 |
| 拆分内容有明确维度标签或引导语 | 0.75 |
| 无多余空行、异常空格、乱码、明显残留特殊字符 | 0.75 |

## 三、易错点（20分）

### 1. 标题层级准确性（5分）

标准层级：

| 层级 | 标准格式 |
|------|----------|
| PSP章节 | `## 一、[工作表名称]` |
| 核心步骤 | `### 第一步：[具体动作或判断]` |
| 模块标题 | `#### 易错点` |
| 易错点主题 | `**◆[易错点主题]**` |
| 具体错误 | `1. [具体错误描述]` |

扣分：

| 问题 | 扣分 |
|------|------|
| `#### 易错点`缺失或写成正文 | 2分/处 |
| 易错点主题未使用 `◆` 但可识别 | 1分/处 |
| 主题下未用 `1.` 列明具体错误 | 1分/处 |
| 全部堆在自然段，无主题和编号 | 本项0分 |

### 2. 覆盖Review Checklist条目（8分）

逐条对照Review Checklist。每遗漏1条RC关键检查项对应的易错点提醒，扣2分；遗漏超过4条，本项0分。

如果Review Checklist本身已经是错误提醒式表达，且可直接作为映射依据，不因语气不是负面句而扣分。

### 3. 内容具体性（7分）

| 问题 | 扣分 |
|------|------|
| 仅写“注意检查”“避免错误”等，未说明检查什么、何种场景发生或后果 | 2分/处 |

## 四、GAAP Difference（5分）

核心要求：GAAP Difference应以标准表格形式呈现，仅对格式进行评分，不对内容准确性进行主观判断。

评分标准：

- GAAP Difference应参考固定资产SOP样例，以Markdown表格列示，至少包含“分类 / 差异项 / PRC / HKFRS”四列：3分。
- 表格结构规范、列对齐正确、无格式混用：2分。
- GAAP Difference未使用表格列示：扣3分。
- GAAP Difference使用表格但缺少“分类 / 差异项 / PRC / HKFRS”中的任一核心列：扣1分/列，扣完为止。
- 完全缺失GAAP Difference章节：本项0分。

## 五、Review Checklist（12分）

### 1. 条目完整性（5分）

检查项应覆盖SOP关键程序、关键判断、证据留痕和复核点。遗漏SOP自身可见关键程序对应检查项，扣1分/处。

### 2. 表格规范性（4分）

- Review Checklist应参考固定资产SOP样例，以Markdown表格列示：2分。
- 表格应至少包含“检查点 / 检查描述”两列，且表头名称与固定资产SOP样例保持一致：1分。
- 每个检查项表述完整，可直接勾选判断（是/否/N/A）：1分。
- Review Checklist未使用表格列示：扣2分。
- 表格缺少“检查点”或“检查描述”列：扣1分/列，扣完为止。
- 每发现1处格式混用或表述不完整：扣1分，扣完为止。

### 3. 可操作性（3分）

检查项应能回答“是/否/N/A”，不得只写笼统原则。笼统不可判断项扣1分/处。

## 六、常用网站（3分）

应包含常用网站章节，并使用表格列示：

- 网站
- 链接
- 主要内容

扣分：

| 问题 | 扣分 |
|------|------|
| 缺少常用网站章节 | 3 |
| 未使用三列表格 | 2 |
| 链接为空、待补充或明显无效 | 1分/处，最多扣3分 |

## 输出证据要求

- 位置必须精确。
- 原文只摘录必要片段。
- 扣分原因必须引用具体规则。
- 整改建议必须可执行。
- RC覆盖检查必须逐条列明或分批列明。
- 重复扣分项合并。
- 长句扣分须说明连接词和独立操作数量。
